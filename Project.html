Project

  • [Samsung SDS's hybrid cloud construction]

    Introduced a cloud system to provide services to multiple affiliates as an integrated resource pool rather than a separate infrastructure. Reduced costs for infrastructure configuration by providing an integrated cloud without building a separate system for each affiliate.


    -Business information-

    -Change the system that was designed with OpenStack and AWS to fit the VMware environment Separation of the business network and the Internet network and change the traffic according to the changed design

    - Hardware installation and configuration for VDI service x86 server, L2 switch, L4 load balancer, SAN switch, SAN storage

    - Installation and configuration of management server for VDI management

    - MS Active Directory, VMware vSphere, Horizon View, MS SQL Server installation Internal NTP server and AD time synchronization Interworking of internal DNS server and AD DNS

    - Create and apply security policies for users' accounts and desktops with domain policies

    - Server configuration for virtual desktop activation and Windows update Windows activation and update of PCs in a business network disconnected from the Internet from KMS and WSUS

    - When a user makes a seat reservation, the user's certificate and user file are automatically copied from the NAS GPKI, a 2 factor authentication method for business network access, and the user profile are copied from the NAX to a public PC
